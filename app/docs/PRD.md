# HarmonyBridge - 项目需求文档 (PRD)

## 1. 项目概述

**项目名称**: HarmonyBridge (和谐之桥)
**定位**: 情侣争端解决与情绪价值 Agent 系统
**核心使命**: 将情绪化的情侣冲突转化为建设性对话，降低争吵频率，缩短情感冲突处理时间。

---

## 2. 核心痛点分析

| 痛点 | 表现 | 影响 |
|------|------|------|
| 情绪脑主导 | 冲突时前额叶皮层被杏仁核压制，失去理性 | 沟通陷入僵局 |
| 缺乏第三方视角 | 双方各执一词，无人客观翻译 | 矛盾螺旋升级 |
| 宣泄口不对位 | 一方需要安慰，另一方却讲道理 | 需求错位导致二次伤害 |
| 诉求表达不清 | 情绪化言语掩盖真实心理动机 | 对方无法准确理解 |

**数据指标**: 系统目标将情感冲突平均处理时间缩短 **60%**。

## 4. 核心 Agent 设计

### 4.1 心理侦测 Agent (PsychDetector)

**职责**: 接收情绪化输入，执行"去情绪化"提取，分析深层心理动机。

**处理流 (CoT)**:
1. **表层情绪识别** — 扫描情绪关键词（委屈、焦虑、失落、愤怒等）
2. **认知语言模式分析** — 检测绝对化词汇("从不"/"总是")、忽视恐惧("不在乎")、倦怠信号("累"/"烦")等
3. **深层需求推导** — 基于语言模式反推心理诉求（安全感、关注、公平感、无条件接纳等）
4. **情绪量化** — 输出效价(-1~1)、唤醒度(0~1)、主导情绪、需求列表、触发因素

**输出示例**:
> 【心理侦测完成】我感知到你话语背后的情绪波长。表面上是焦虑，深层需求其实是安全感和被爱感。那些尖锐的措辞，其实是保护内心柔软的方式。

### 4.2 共情疏解 Agent (EmpathySoother)

**职责**: 为情绪波动方提供即时心理按摩与压力释放引导。

**处理流 (CoT)**:
1. **情绪共鸣定位** — 匹配共情模板（委屈→"你受了委屈却还要强撑"；愤怒→"愤怒是内心在喊'这对我很重要'"）
2. **压力指数评估** — 基于唤醒度评估压力等级(1~10)
3. **定制化疏解策略** — 高唤醒→呼吸调节指导；低唤醒→认知重构+需求确认

**输出示例**:
> 焦虑像一只紧紧攥住心脏的手。但请记住，焦虑的另一面是你对这段关系的在意。深呼吸，吸气4秒，屏息4秒，呼气6秒，我在这里陪着你。

### 4.3 外交官 Agent (Diplomat)

**职责**: 作为中立第三方，将诉求转化为非暴力沟通(NVC)语言，提供破冰建议。

**处理流 (CoT)**:
1. **攻击性语言剥离** — 将"你陈述"转化为"我陈述"，消除指责感
2. **NVC四要素重构** — 观察→感受→需求→请求
3. **破冰策略匹配** — 基于双方情绪状态匹配破冰方案（温柔重启、错位道歉、需求坦白、暂停邀约、翻译转述）
4. **双视角整合** — 确保建议不侵犯对方边界，同时兼顾双方需求

**输出示例**:
> 【外交官翻译】我注意到你最近很晚回家...，我感到不安，因为我需要安全感，你是否愿意和我一起找找让我们都舒服的方式？
> 【破冰建议 · 需求坦白】直接说出你的脆弱："其实我刚刚那样说，是因为我怕你觉得我不重要。"

## 5. WebSocket 通讯协议

### 5.1 消息类型

**客户端 → 服务器**:
- `join` — 加入调解室（携带 role + name）
- `chat` — 发送消息内容
- `typing` — 输入状态
- `session_end` — 结束会话

**服务器 → 客户端 (Broadcast)**:
- `user_joined` — 用户加入通知
- `message` — 新消息（用户/Agent/系统）
- `agent_thinking` — Agent 开始思考（CoT 步骤预览）
- `agent_complete` — Agent 完成分析（含完整结果）
- `emotion_update` — 情绪数据更新
- `stage_change` — 会话阶段变更
- `session_state` — 完整会话状态同步

### 5.2 会话阶段机

```
waiting → intake → analysis → empathy → diplomacy → resolution → closed
 等待      收录       侦测        疏解        调解        方案        结束
```

## 7. 数据模型

### 7.1 核心类型

```typescript
interface User {
  id: string;
  role: 'partnerA' | 'partnerB';
  name: string;
  emotionScore: number;
}

interface Message {
  id: string;
  senderId: string;
  senderRole: UserRole | 'system' | AgentType;
  senderName: string;
  content: string;
  timestamp: number;
  type: 'user' | 'agent_response' | 'system';
  agentType?: AgentType;
  emotionData?: EmotionData;
  cotSteps?: CoTStep[];  // 思维链步骤
}

interface EmotionData {
  valence: number;      // 效价 -1~1
  arousal: number;      // 唤醒度 0~1
  dominantEmotion: string;
  needs: string[];
  triggers: string[];
}

interface CoTStep {
  agent: AgentType;
  step: number;
  thought: string;
  action: string;
}
```
